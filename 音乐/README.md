# 音乐（Music）知识库组织说明（面向对象 / 可扩展 & 速查）

这部分按“面向对象/类型系统”的方式组织：
- **结构（structures）**：音乐对象的“类型定义”（体裁/形式/乐器…）
- **模块（modules）**：跨类型复用的能力（旋律/和声/节奏/音色/编曲/混音…）
- **作品（works）**：具体作品卡片（歌/专辑/现场）
- **艺人（artists）**：创作者卡片（作曲/编曲/演奏/制作）
- **听歌记录（listening）**：时间轴与收藏入口（更偏“日志/收藏夹”）

## 命名约定（让文件名自带层级信息）
内容页尽量用前缀体现“它属于哪一层”：
- 体裁：`音乐-体裁-xxx.md`
- 形式：`音乐-形式-xxx.md`
- 乐器：`音乐-乐器-xxx.md`
- 模块：`音乐-模块-xxx.md`
- 作品：`音乐-作品-歌曲-xxx.md` / `音乐-作品-专辑-xxx.md`
- 艺人：`音乐-艺人-xxx.md`
- 记录：`音乐-听歌记录-YYYY-MM.md`

（文件放在哪个目录仍然按 `structures/modules/works/...` 来，但文件名本身也能看出层级。）
